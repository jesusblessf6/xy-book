# XY-Book — 数据采集与清洗

> 本文档描述数据采集和清洗的流程。数据输入格式和结构详见 DATA_INPUT.md。
> DATA_INPUT.md 是数据格式的权威文档，本文档只描述采集和清洗方法。

## 数据源

| 平台 | 采集内容 | 方式 |
|------|---------|------|
| 微博 | 热搜评论、大V评论区 | 手动/API |
| 抖音 | 热门视频评论 | 手动/爬虫 |
| 知乎 | 问题回答、评论区 | 手动/API |
| B站 | 视频评论、弹幕 | 手动/API |
| 小红书 | 笔记评论区 | 手动 |

## 采集原则

1. **按用户聚合** — 采集的核心是同一用户在不同帖子下的多条评论，这样才能看出完整行为模式
2. **附上帖子上下文** — 每条评论必须附上对应的帖子内容，否则 LLM 无法理解评论语境
3. **脱敏处理** — 不采集真实用户名、头像、ID，用脱敏 ID 代替
4. **只记录可观测的互动** — 评论、点赞、转发是可观测的，"浏览了但没评论"不可观测（见 INTEREST_INFERENCE.md）

## 清洗流程

```
原始采集数据
  │
  ├─ 1. 去 bot — 识别并移除明显的机器人/水军账号
  │     - 重复内容检测
  │     - 发帖频率异常
  │     - 内容与上下文无关
  │
  ├─ 2. 去广告 — 移除广告、推广内容
  │
  ├─ 3. 去重复 — 移除完全相同的重复评论
  │
  ├─ 4. 格式标准化 — 统一到 DATA_INPUT.md 定义的格式
  │
  ├─ 5. 按用户聚合 — 将同一用户的评论合并到一个用户文件中
  │
  └─ 6. 验证 — 每个用户至少 3 条评论，帖子上下文完整
```

## 清洗后的存储

清洗后的数据按 DATA_INPUT.md 定义的格式存储：

```
data/inputs/batch-001/
├── meta.yml                  # 批次元信息
└── users/
    ├── u_0001.yml            # 用户1的所有评论+帖子上下文
    ├── u_0002.yml            # 用户2
    └── ...
```

## 技术约束

- **不引入向量数据库** — 特征矩阵在内存中计算，HDBSCAN 聚类用 numpy/scipy
- **文件优先** — 数据以 YAML/JSON 文件存储，用 ripgrep 搜索，不用向量索引
- **特征提取用 LLM** — 评论的说话风格、立场、情绪特征由 LLM 分析提取
- **聚类用特征矩阵** — LLM 提取的特征转为数值向量，送入 HDBSCAN 聚类，全程在内存中完成
